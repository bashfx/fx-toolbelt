1. FUNCTION ORDINALITY HIERARCHY
--------------------------------

The foundation of all BashFX code organization follows a strict ordinality model:

**Entry Level (Script Execution)**
- Script entry point via main() invocation
- Global scope initialization and environment setup

**Super-Ordinal Functions**
- main()     : Primary entrypoint, orchestrates lifecycle
- dispatch() : Command router executing do_* functions  
- options()  : Super-ordinal argument parser, sets opt_* variables

**High-Order Functions (Dispatchable)**
- do_*       : Primary user-facing commands called by dispatch
- bookdb_*   : Vanity-prefixed dispatchable functions
- Public functions managing user interaction and guards

**Mid-Level Helpers**
- _function  : Standard subroutines (single underscore)
- Helper functions breaking down complex logic

**Low-Level Literals**  
- __function : "Close to the metal" functions (double underscore)
- Single-purpose, raw operations trusting inputs

**Guard Functions**
- is_*       : State validation functions (e.g., is_file, is_dir)
- dev_*      : Development functions with user-level guards

CRITICAL RULE: User-level error handling ONLY at High-Order level. Low-level functions handle system errors only.

2. STANDARD INTERFACE COMPONENTS
-------------------------------

**Predictable Variables (Lazy Naming)**
- Status: ret, res (return codes, results)  
- Strings: str, msg, lbl
- Paths: src, dest, path
- Collections: arr, grp, list
- Identity: this, that, ref, self
- Iterators: i, j, k
- Spatial: x, y, z  
- Comparison: a, b, c
- Logic: p, q, r

**Standard Function Roster**
- main()     : Super-ordinal lifecycle orchestrator
- options()  : Super-ordinal argument resolver  
- dispatch() : Super-ordinal command router
- status()   : High-order environment/state reporter
- usage()    : High-order help text display
- version()  : High-order version/logo display
- is_*()     : Guard functions for state validation

**Variable Scoping Convention**
- ALL_CAPS   : Configuration/pseudo-constants from session/settings
- lowercase  : Ephemeral local scope (function args, locals)
- Example: OPT_DEBUG (framework setting) vs opt_debug (local state)

4. BUILD.SH PATTERN v1
---------------------

**For scripts >1000 lines, implement parts-based building:**

**Directory Structure**
```
project/
├── parts/
│   ├── build.map          # Ordered part definitions
│   ├── 01_header.sh       # Shebang, meta, portable declarations
│   ├── 02_config.sh       # Configuration variables, XDG paths
│   ├── 03_stderr.sh       # Logging functions (or include)
│   ├── 04_helpers.sh      # Helper functions
│   ├── 05_api.sh          # Dispatchable functions
│   ├── 08_main.sh         # Main function and options
│   └── 09_footer.sh       # Main invocation
├── build.sh               # Build orchestrator
└── target_script.sh       # Generated output (don't edit directly)
```

**Build.map Format**
```bash
# Build Map  
# Format: NN : target_filename.sh
01 : 01_header.sh
02 : 02_config.sh  
03 : 03_stderr.sh
# ... additional parts
```

**Critical Guidelines**
- Part files must end on complete functions (no split code)
- Only 01_header.sh should contain shebang line
- Individual parts should be <300-500 lines
- Scripts approaching 3000-4000 lines should be ported to RSB (Rust)

5. DEPLOYMENT & INSTALLATION PATTERNS
------------------------------------

**XDG+ Installation Standard**
- Script packages install to: `$HOME/.local/lib/fx/appname/`
- Executables symlinked to: `$HOME/.local/bin/` (no .sh extension)
- Configuration in: `$HOME/.local/etc/appname/`
- Data files in: `$HOME/.local/data/appname/`
- Cache/tmp in: `$HOME/.cache/tmp/` or project-local `.tmp/`

**Rewindable Operations**
- Every install must have corresponding uninstall
- Sentinels track file modifications for safe rollback
- RC files manage session state via linking patterns
- File operations use backup-before-modify approach

**Three Dispatcher Patterns:**

```bash

**B. Scoped Commands (Entity-Specific Operations)**

```
*Use when 3+ operations exist only within one entity's scope*

**C. Predicate Commands (Cross-Entity Operations)**
```bash

tool rotate project    # Same conceptual operation

```
*Use when same conceptual operation applies to multiple entity types*

**Anti-Pattern: Hyphenated Commands**
```bash
# AVOID: Creates collision-prone command surface
tool rotate-master
tool deep-scan-files  
# PREFER: Natural language patterns with proper scoping
tool rotate master
tool scan files --deep
```

7. OPTIONS & ARGUMENT PARSING
----------------------------

**Standard Flags Implementation**
```bash
options(){
  local this next opts=("${@}");
  for ((i=0; i<${#opts[@]}; i++)); do
    this=${opts[i]}
    case "$this" in
      --debug|-d)    opt_debug=0; opt_quiet=1 ;;
      --trace|-t)    opt_trace=0; opt_debug=0 ;;  
      --quiet|-q)    opt_quiet=0 ;;
      --force|-f)    opt_force=0 ;;
      --yes|-y)      opt_yes=0 ;;
      --dev|-D)      opt_dev=0; opt_debug=0; opt_trace=0 ;;
      --file=*)      opt_file_path="${this#*=}" ;;
      -*)            error "Invalid flag [$this]" ;;
    esac
  done
}
```

**Hierarchical Verbosity Logic**
- Higher levels enable lower levels automatically
- -D (dev) enables -d (debug) and -t (trace)
- -t (trace) enables -d (debug)  
- -q (quiet) overrides all verbosity flags

10. XDG+ STANDARD (XDG VERSION 1)
---------------------------------

**BashFX XDG+ Hierarchy**
```bash
$HOME/.local/           # XDG_HOME (base for all BashFX paths)
├── lib/               # XDG_LIB_HOME (script packages)
│   └── fx/           
│       └── appname/   # App-specific library location
├── etc/               # XDG_ETC_HOME (configuration, preferred over ~/.config)
├── data/              # XDG_DATA_HOME (data libraries, databases, refs) 
├── bin/               # XDG_BIN_HOME (symlinked executables, flat namespace)
└── state/             # XDG_STATE_HOME (runtime state files)

$HOME/.cache/
└── tmp/               # XDG_TMP_HOME (local tmp instead of /tmp)
```

**XDG+ Helper Functions**
```bash
xdg_path(){           # Get XDG+ compliant path
  case "$1" in
    (home) echo "$HOME/.local" ;;
    (lib)  echo "$HOME/.local/lib" ;;
    (etc)  echo "$HOME/.local/etc" ;;  
    (bin)  echo "$HOME/.local/bin" ;;
    (data) echo "$HOME/.local/data" ;;
    (tmp)  echo "$HOME/.cache/tmp" ;;
  esac
}

xdg_type(){           # Determine XDG+ type from path
xdg_init(){           # Create XDG+ directory if needed

```
